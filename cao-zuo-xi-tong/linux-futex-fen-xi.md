# linux futex 分析

在 linux 并发编程领域，futex (fast userspace mutex）是非常核心且基础的概念。几乎所有的现代高级同步原语（c++ std::mutex go sync.Mutex posix pthread\_mutex）在 linux 上的底层实现都依赖于它。

#### 1 为什么需要 futex

在 futex 诞生之前，系统主要通过传统的 system v ipc 或早期的方式来实现锁。这些早期实现的致命缺陷是：无论锁是否发生竞争，都需要执行系统调用进入内核态。

* 痛点：系统调用的开销是很昂贵的
* 现实情况：在实际的程序中，大部分时候获取锁的过程是没有竞争的，如果为了一个没有竞争的锁没次都进出内核，会浪费cpu 资源

为了解决这个问题，futex 被引入，它的核心设计理念就是：在无竞争的情况下，完全在用户态；只有发生竞争时才进入内核态。

#### 2 futex 工作原理

futex 是由两部分组成的：用户态的一个整型变量 + 内核态的等待队列

它的工作流程可以分为“快速路径”和“慢速路径”。

* 快速路径--完全在用户态\
  当线程想要获取锁时，首先使用CAS操作去一个内存地址尝试修改
  * 如果修改成功，线程直接获取锁，继续执行，整个过程不涉及系统调用，速度极快
  * 释放锁时，通过原子操作将变量改回初始值
* 慢速路径--陷入内核态\
  如果CAS失败，说明锁已经被其他线程占用了，这时
  * 线程调用 futex，告诉内核把它挂起在这个地址的队列上
  * 内核将线程放在一个该内存地址相关的等待队列中，并将线程让出 cpu
  * 当占用锁的线程处理完，他在用户态释放锁，会发现有其他线程等待，此时他会调用 futex 去唤醒在内核中等待的线程

#### 3 futex 的接口

```
int futex(int *uaddr, int futex_op, int val, const struct timespec *timeout, ...);
```

* FUTEX\_WAIT
  * 语义：如果用户态地址 uaddr 的值等于 val，就让线程休眠等待，直到有人唤醒
  * 细节：条件判断和休眠操作是由内核保证原子性，不会出现“刚检查值准备休眠，另一个线程来释放锁发送了唤醒信号导致永久休眠”
* FUTEX\_WAKE:
  * 语义：“唤醒等待在地址 uaddr 上最多 val 个线程”，val = 1唤醒一个，val=INT\_MAX  唤醒所有
