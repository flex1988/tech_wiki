# Ceph: A Scalable, High-Performance Distributed File System

## 摘要

我们开发了Ceph，一个具有优秀的性能，可靠性，可扩展性的分布式文件系统。Ceph 通过将分配表替换为一种专为由不可靠对象存储设备（OSD）组成的异构和动态集群而设计的伪随机数据分布函数（CRUSH），最大化地实现了数据与元数据管理的分离。我们通过将数据复制，故障检测和恢复任务分发给运行着专用本地文件系统的 OSD，增加了 OSD 的智能化。一个动态的分布式元数据集群提供了及其有效率的元数据管理，并无缝适应广泛的通用和科学计算文件系统工作负载。性能测量显示在多种工作负载下，ceph有极其优秀的io性能和可扩展的元数据管理，可以支持元数据操作达到 250k/s。

### 1. 介绍

系统设计者一直以来都想提高文件系统的性能，因为它对于整个系统的性能非常关键。特别是科学界和高性能计算社区，推动了分布式存储系统在性能和可扩展性方面的进步，通常能提前几年预示更通用的需求。传统的解决方案，典型如 NFS，提供了一个非常直观的模型，在这个模型中 server 导出一个文件系统层次结构，客户端可以将其映射到本地的命名空间中。尽管应用非常广泛，client/server 模型固有的集中化已被证明是实现可扩展性的重大障碍。

更近期的分布式文件系统架构基于对象存储，原有的机械盘被智能的对象存储设备替代了，OSD 不仅仅是一块盘，它还包括了 CPU，网络和本地缓存包括底层的磁盘或者是RAID。OSD把传统的块层的接口替换成了可以读写更大的变长字节区间也就是对象，把底层块的分配交给了设备自己。client 通常和 metadata server 交互来执行 metadata 操作（open/rename），然后和 OSD 交互来读写，显著的提高了可扩展性。

系统在采取了这种模型后，还继续受到了可扩展性的影响，由于meta data 的分布式的限制。对于传统文件系统的持续依赖，比如分配列表和inode table，以及不愿意将只能下放给 OSD，进一步限制了可扩展性和性能，并增加了可靠性成本。

我们提出了 Ceph，一个分布式文件系统，它提供了卓越的性能和可靠性，同时承诺了无与伦比的可扩展性。我们的架构基于这样一个假设：PB级规模的系统本质上是动态的：大型系统不可避免地是增量构建的，节点故障是常态而非特例，并且工作负载的质量和特征会随着时间的推移而不断发生变化。

Ceph 通过消除文件分配表并将其替换为生成式函数，实现了数据和元数据的解耦。这使得 Ceph 能够利用 OSD 具备的智能特性，将围绕数据访问、更新序列化、数据复制与可靠性、故障检测及数据恢复等方面的复杂性下放到各个节点中。Ceph 采用了一种高度自适应的分布式元数据集群架构，极大地提升了元数据访问的可扩展性，并由此提高了整个系统的可扩展性。我们讨论了驱动我们在架构设计中做出诸多选择的目标与工作负载假设，分析了这些选择对系统可扩展性和性能的影响，并分享了我们在实现功能性系统原型过程中的经验。
